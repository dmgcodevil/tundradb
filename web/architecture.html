String values use a tiered StringArena. Short strings (â‰¤16, â‰¤32, â‰¤64 bytes) go into fixed-size pools, giving predictable allocation sizes. Longer strings fall back to heap allocation. All strings are stored as StringRef (pointer + length).

Why arenas? Graph databases create millions of small, long-lived objects. Traditional new/delete causes fragmentation and cache misses. Arenas allocate in large contiguous blocks, improving locality and enabling bulk deallocation.
